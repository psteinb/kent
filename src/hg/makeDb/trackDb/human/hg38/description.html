IUPAC characters
In addition to A, C, G, T and N, the raw sequence files from NCBI also include additional IUPAC ambiguity codes for bases that could not be positively identified as A, C, G or T (see NCBI or bioinformatics.org or Wikipedia for more information) (***WE SHOULD CHOOSE ONE OF THESE REFERENCES***). As part of the UCSC assembly creation process, these IUPAC characters are converted to N's, so the FASTA files available for download from UCSC contain N's instead of the original IUPAC characters. For FASTA files containing these IUPAC characters, the raw data files are available for download from the NCBI FTP site.

Note on chrM
The hg19 assembly was released with the mitochondrion sequence (represented as "chrM" in the Genome Browser) contained in GenBank record NC_001807. After the release of hg19, the mitochondrion sequence in NC_001807 was replaced with record NC_012920, though the hg19 assembly was not updated to reflect this change. The hg38 assembly now includes this newer mitochondrion sequence. Please see the Revised Cambridge Reference Sequence (rCRS) page for more information.

Chromosome naming scheme
With the release of hg38, there has been a slight change to the UCSC chromosome naming scheme:

  • Haplotype chromosome, unplaced contig and unlocalized contig names now include their NCBI accession number (e.g., chr6_GL000256v2_alt)
  • The "v2" at the end of the accession number indicates the NCBI version number
  • Haplotype chromosome names consist of the chromosome number, followed by the NCBI accession number, followed by "alt"
  • Unplaced contig names consist of the chromosome number, followed by the NCBI accession number, followed by "random"
  • Unlocalized contig names (contigs whose associated chromosome is not known) consist of "chrUn" followed by the NCBI accession number
