Introduction

The Custom Composite Builder is a tool which provides a way to build a custom composite track grouping of wiggles that can be sorted, displayed in a variety of ways, and saved in a session for sharing or later viewing. Wiggles can be collected from disparate sources including track hubs, custom tracks, and native tracks into one central location based on differing dimensions of metadata, or any other criteria that is useful.

A composite track is a grouping for a set of similar-type annotation tracks. A set of tracks within a composite track are referred to as "sub-tracks."

Key Features

  • The Custom Composite Builder builds composites for a single genome assembly.
  • Wiggles can be copied into collections from a tree view with a list of all the available wiggles, or added individually from the genome browser.
  • A single wiggle track can occur multiple times in a collection and in multiple collections so data can be viewed with multiple viewing options at the same time.
  • Wiggles in a custom composite can be displayed with multiwig overlay methods (stacked, transparent, etc.)
  • Wiggles in a custom composite can be sorted by similarity or expression in the genome browser.
  • Simple math (addition and subtraction) can be done on wiggles in a collection.
  • Custom composites have user-defined names and descriptions.
  • Wiggle tracks within custom composites can be renamed and new descriptions added.
  • All wiggle types are supported: bigWig, wig, and bedgraph.
  • Tracks from trackhubs can be collected and displayed without connecting to the hubs.
